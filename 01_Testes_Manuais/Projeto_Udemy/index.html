🚨 Bugs de Maior Impacto

Abaixo destaco as falhas mais importantes encontradas durante os testes, que afetam a experiência do usuário na pesquisa de cursos.

QAUDEMY-13 Prioridade: Alta
Relacionado ao CT.05

Erro 404 ao fazer pesquisa em branco

Descrição: Ao tentar pesquisar sem digitar nenhum termo, o site adiciona a palavra 'undefined' no endereço da página (URL). Isso encaminha o usuário para uma página de erro 404.

🔬 Inspeção via DevTools

Uso da aba Network (Rede) para identificar a requisição incorreta:

GET https://www.udemy.com/courses/search/undefined/ Status: 404 (Not Found)
Comportamento:
  • Esperado: O sistema não deveria buscar ou deveria pedir um termo válido.
  • Atual: Redireciona para uma página que não existe (.../undefined).

🎯 O que fiz neste projeto

Testes Funcionais

Testei a plataforma como um usuário real, explorando menus e inserindo dados inválidos para ver como o sistema reagia aos erros.

Registo de Bugs

Documentei os erros encontrados com passo-a-passo claro e evidências visuais (prints e vídeos) para facilitar a correção pelos desenvolvedores.

Uso de DevTools

Usei a ferramenta de inspeção do navegador (aba Network) para identificar se os erros visuais vinham de links quebrados ou falhas na requisição.

Documentação

Criação de Casos de Teste estruturados e elaboração de relatórios resumindo os resultados da execução.

📊 Resumo dos Resultados e Ambiente

  • Dos 12 cenários planejados que testei, encontrei 6 bugs (taxa de falha de 50%).
  • Foquei bastante em validar se os filtros de pesquisa funcionavam bem, para garantir que o usuário não se perca buscando cursos irrelevantes.
  • Os testes foram executados no navegador Google Chrome (Modo Anônimo), para garantir que o cache antigo não interferisse nos resultados do teste.
